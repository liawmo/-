# 五子棋小游戏

一个基于原生 HTML、CSS 和 JavaScript 实现的五子棋小游戏。项目不依赖前端框架，直接打开 `index.html` 可以进行本机双人对弈；如果需要局域网联机，需要通过 `server.js` 启动本地服务。

## 项目说明

当前版本使用 Canvas 绘制 15 x 15 棋盘，并在前端维护棋局状态。黑棋先手，双方轮流落子，任意一方在横向、纵向或斜向连续形成五子即获胜。棋盘落满且无人获胜时判定为平局。

主要功能包括：

- 15 x 15 标准棋盘
- 黑白双方本地轮流落子
- 局域网房间联机对战
- 鼠标悬停预览当前落点
- 移动端触摸落子
- 自动判断五子连线胜负
- 自由规则和标准规则切换
- 标准规则下支持黑棋长连、四四、三三禁手判断
- 普通、障碍棋盘、限时快棋、随机开局四种游戏模式
- 显示当前回合、胜负状态、手数和限时倒计时
- 标记最后一步落子
- 使用淡金色发光轨迹高亮胜利连线
- 对局结束弹层和再来一局入口
- 支持按回合悔棋
- 支持重新开始
- 棋谱记录
- 比分统计和清空统计
- 棋局 JSON 导入导出
- 木纹、简洁、深色主题切换
- 使用 `localStorage` 保存当前棋局和比分
- 响应式布局，适配桌面和移动端屏幕

## 游戏模式

1. 普通模式  
   基础五子棋玩法，双方轮流落子，先形成五连的一方获胜。

2. 障碍棋盘  
   开局随机生成 12 个障碍点。障碍点不可落子，会改变棋盘空间和进攻路线。

3. 限时快棋  
   每手 15 秒。当前玩家超时后自动判负，适合更快节奏的短局。

4. 随机开局  
   开局自动落下黑白各两手，每局从不同局面开始，减少重复开局。

## 规则说明

- 自由规则：五子或更多连成一线即获胜。
- 标准规则：白棋五子或更多连成一线获胜；黑棋会检查长连、四四、三三禁手。
- 当前禁手判断采用适合本项目的轻量实现，已经覆盖常见长连、双四、双三形态。若要完全对齐职业连珠规则，后续仍建议继续补充更系统的规则测试。

## 本机运行

直接用浏览器打开：

```text
D:\Code\gomoku\index.html
```

这种方式适合本机双人对弈，但不能使用局域网房间联机。

## 局域网联机

局域网对战需要有一台设备作为“主机”运行项目内置的 Node 服务，不能只靠双击 `index.html` 完成。原因是直接打开文件时页面运行在 `file://` 环境里，只能执行浏览器里的前端代码，不能创建房间服务，也不能让其他设备访问这台设备上的本地文件。

只有主机需要安装 Node.js；其他手机、平板、电脑不需要安装 Node，只要能打开浏览器并连接到同一个 Wi-Fi 或局域网即可。

在项目目录运行：

```bash
node server.js
```

Windows 也可以直接双击项目里的：

```text
start-lan.bat
```

启动后终端会显示本机访问地址和局域网访问地址，例如：

```text
本机访问: http://localhost:3000
局域网访问: http://192.168.1.23:3000
```

如果打不开，按下面顺序排查：

1. 主机自己先打开 `http://localhost:3000`。如果打不开，说明服务没有启动成功，需要查看终端里的报错。
2. 如果提示 `node` 不是内部或外部命令，说明主机没有安装 Node.js。需要在主机安装 Node，或换一台有 Node 的设备作为主机。
3. 如果提示端口被占用，可以换端口启动，例如 Windows PowerShell：

```powershell
$env:PORT = "3001"
node server.js
```

4. 如果主机能打开 `http://localhost:3000`，但其他设备打不开 `http://192.168.x.x:3000`，通常是 Windows 防火墙拦截。请允许 Node.js 通过“专用网络”，并确认两台设备在同一个 Wi-Fi 下。
5. 如果终端显示多个局域网地址，优先尝试和手机同网段的地址。例如手机 IP 是 `192.168.1.50`，就优先尝试 `192.168.1.x:3000`，不要选 `172.x.x.x` 或虚拟网卡地址。

使用方式：

1. 两台设备连接到同一个 Wi-Fi 或同一个局域网。
2. 选择其中一台有 Node.js 的设备作为主机，运行 `node server.js`。
3. 主机和其他设备都打开终端显示的局域网访问地址，例如 `http://192.168.1.23:3000`。
4. 页面通过 HTTP 地址打开后，“局域网对战”区域会自动显示联机可用；此时点击“创建房间”或“加入”即可切换到局域网对战。
5. 第一位玩家点击“创建房间”，复制房间码。
6. 第二位玩家输入房间码并点击“加入”。
7. 创建房间者执黑，加入者执白。

联机说明：

- 局域网联机使用 HTTP 轮询同步棋局，适合家庭、宿舍、教室等小范围网络。
- 前端会自动使用当前页面所在地址的 `/api` 服务。也就是说，用 `http://192.168.1.23:3000` 打开页面时，不需要在网页里额外填写服务器地址。
- 创建房间的一方执黑，可以重开棋局和更改模式。
- 加入房间的一方执白，只能在轮到白棋时落子。
- 落子、悔棋、重开、规则和主题变化会同步到同房间另一端。
- 没有 Node 的设备可以作为客户端加入对战，但不能作为主机创建局域网服务。
- 如果手机无法访问主机地址，需要检查电脑防火墙是否允许 Node.js 访问局域网。
- 蓝牙互联暂未实现。浏览器的 Web Bluetooth 更适合连接外设，不适合稳定承载双人棋局同步。

## Android APK

项目已有 Android WebView 封装工程，路径为 `android/`。当前生成的可安装包位置：

```text
D:\Code\gomoku\gomoku-android16-debug.apk
```

这个 APK 使用本地 WebView 离线加载 `index.html`、`game.js` 和 `styles.css`，不联网也能游玩。当前包使用 debug 签名，适合自己手机安装测试；如果后续要发布到应用商店，需要再生成 release 签名包。

如果需要重新构建，可以使用项目内的临时构建工具：

```powershell
$env:JAVA_HOME = "D:\Code\gomoku\.android-build\jdk\<jdk目录>"
$env:ANDROID_HOME = "D:\Code\gomoku\.android-build\android-sdk"
$env:ANDROID_SDK_ROOT = "D:\Code\gomoku\.android-build\android-sdk"
D:\Code\gomoku\.android-build\gradle\<gradle目录>\bin\gradle.bat -p android assembleDebug
```

## 后续可优化内容

### 高优先级

1. 为禁手规则补测试  
   当前已经加入三三、四四、长连禁手，但禁手规则天然复杂。建议补充一组典型棋型测试，避免误判或漏判。

2. 拆分棋局逻辑  
   现在所有逻辑仍集中在 `game.js` 中。后续可以拆分为棋盘绘制、棋局状态、规则判断、模式初始化、联机同步、存档恢复、事件绑定等模块，方便维护和扩展。

3. 优化联机稳定性  
   当前局域网联机使用 HTTP 轮询。后续可以升级为 WebSocket，降低延迟，并更好地处理断线重连、房间过期和观战。

### 中优先级

4. 优化模式组合  
   当前一次只能选择一种游戏模式。后续可以把“限时”做成独立开关，让限时可以和障碍棋盘、随机开局组合使用。

5. 增强棋谱能力  
   当前棋谱只展示落子顺序和坐标。后续可以增加棋谱回放、跳转到指定手数、复制棋谱文本和导出更通用的棋谱格式。

6. 增加更多趣味模式  
   可以继续加入小棋盘快棋、指定方向获胜、目标任务、连胜挑战等模式，但需要控制复杂度，避免规则变得难以理解。

### 低优先级

7. 增加音效和动效设置  
   落子、悔棋、胜利、超时等操作可以加入轻量反馈，并提供开关，避免影响不需要音效的用户。

8. 增加无障碍支持  
   当前主要依赖鼠标和触摸操作。后续可以补充键盘落子、焦点状态、读屏提示和更明确的 ARIA 文案。

## 新想法：猫狗五子棋

可以保留传统黑白圆形棋子，在胜利时加入更有角色感或仪式感的视觉反馈，例如淡金色胜利光轨、获胜方头像弹层，或只让获胜五连棋子出现轻量徽章。

可扩展表现：

- 胜利五连使用淡金色发光轨迹标记，避免整盘棋子过度装饰。
- 白方获胜时，可以在结果弹层或局部徽章里展示加菲猫风格表情。
- 黑方获胜时，可以在结果弹层或局部徽章里展示边牧风格表情。
- 落子时加入轻量反馈，让棋子更有手感，但普通棋子仍保持黑白圆形。
